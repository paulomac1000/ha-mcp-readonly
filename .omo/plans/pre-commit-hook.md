# Pre-commit Hook â€” CI Gate at Commit Time

## TL;DR

> **Quick Summary**: Create `.pre-commit-config.yaml` and installable git hook that runs the same checks as CI (ruff, mypy, bandit, CAFDS, version sync, semgrep, unit tests) BEFORE commits, preventing CI failures from reaching the remote.
>
> **Deliverables**:
> - `.pre-commit-config.yaml` â€” declarative hook config
> - `.git/hooks/pre-commit` â€” executable hook (generated by `pre-commit install`)
> - `AGENTS.md` â€” updated with pre-commit agent instructions
>
> **Estimated Effort**: Quick (~15 min)

---

## Why

CI caught failures AFTER push. The pre-commit hook must catch them BEFORE push so that:
1. Agents don't push broken code
2. CI doesn't waste runner minutes on lint failures
3. Developers get instant feedback on their machine

## Implementation

### `.pre-commit-config.yaml`

```yaml
repos:
  - repo: local
    hooks:
      - id: ruff-check
        name: Ruff check
        entry: ruff check .
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]

      - id: ruff-format
        name: Ruff format check
        entry: ruff format --check .
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]

      - id: mypy
        name: mypy strict
        entry: mypy tools/ --strict
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]

      - id: bandit
        name: Bandit security scan
        entry: bandit -r tools/ -ll
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]

      - id: version-check
        name: Version consistency
        entry: bash -c 'V1=$(grep __version__ version.py | cut -d\" -f2); V2=$(grep -m1 "^version = " pyproject.toml | cut -d\" -f2); [ "$V1" = "$V2" ] || { echo "ERROR: version.py=$V1 != pyproject.toml=$V2"; exit 1; }'
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]

      - id: cafds-docs
        name: CAFDS documentation validation
        entry: bash -c 'if [ -f afds_config.yaml ] && command -v curl &>/dev/null; then curl -sS https://raw.githubusercontent.com/paulomac1000/ai-skills/main/skills/afds-doc-writer/docs_validate.py | python3 - --config afds_config.yaml --baseline .afds-baseline.json ./ || exit 1; else echo "CAFDS validation skipped (no config or curl)"; fi'
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]

      - id: semgrep
        name: Semgrep security scan
        entry: bash -c 'if command -v semgrep &>/dev/null; then semgrep --config=p/auto --config=p/secrets --config=p/owasp-top-ten --error --quiet . || exit 1; else echo "Semgrep not installed â€” skipping (install: pip install semgrep)"; fi'
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]

      - id: unit-tests
        name: Unit tests
        entry: bash -c 'python -m pytest tests/unit/ -q --tb=short || exit 1'
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-commit]
```

### AGENTS.md Addition

```markdown
## Pre-commit Hook (MANDATORY for all commits)

This project uses a pre-commit hook that runs the SAME checks as CI BEFORE every commit:

```bash
pre-commit install
```

**What the hook runs (in order):**
1. `ruff check .` + `ruff format --check .`
2. `mypy tools/ --strict`
3. `bandit -r tools/ -ll`
4. `version.py` â†” `pyproject.toml` consistency check
5. CAFDS documentation validation (curl from GitHub)
6. Semgrep security scan (p/auto + p/secrets + p/owasp-top-ten)
7. `pytest tests/unit/ -q`

**Agent workflow before committing:**
1. Make your changes
2. Run `pre-commit run --all-files` to verify everything passes
3. If any hook fails, fix the issue BEFORE committing
4. Commit after all hooks pass

**Installation (one-time):**
```bash
pip install pre-commit semgrep
pre-commit install
```

**CI badge**: If pre-commit passes, CI WILL pass (they run the same checks).
**If you skip the hook**: Your commit WILL be rejected by CI. Fix at commit time, not at PR time.
```
